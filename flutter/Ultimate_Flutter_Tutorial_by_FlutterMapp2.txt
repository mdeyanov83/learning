üó∫Ô∏è 7. Navigation
    Navigator() is a Flutter widget and class that manages a stack of routes (screens). It lets you navigate
    between pages, push new screens, and pop back to previous ones ‚Äî like a web browser‚Äôs history.
    * Stack-based navigation:
        - Each screen is a route.
        - You can push (go forward) or pop (go back).

    Navigator.push(
        context,
        MaterialPageRoute(builder: (context) => SecondPage()),
        );
    Navigator.pop(context);
    üß± Common methods:
        push()	Opens a new route (screen) and adds it to the stack
        pop()	Closes the current route and goes back
        pushReplacement()	Replaces the current route with a new one
        pushNamed() / popNamed()	Navigate using route names (defined in MaterialApp)
        canPop()	Checks if there‚Äôs a route to go back to
    üìå Tip:
        Use Navigator.pushNamed() for cleaner navigation when you have multiple screens.
        For modern navigation, consider go_router or Navigator 2.0 (Router API) for complex apps.
        Navigator works perfectly with Scaffold for building multi-screen apps.

    02:02:34 Push
        Navigator.push() is used to navigate to a new screen (route) by pushing it onto the navigation stack.
        It shows the new screen on top of the current one, allowing the user to go back later with Navigator.pop().
        Navigator.push(
            context,
            MaterialPageRoute(builder: (context) => NextPage()),
        );
        üß± Parameters:
            context ‚Üí The current widget‚Äôs build context.
            route ‚Üí The route (page/screen) to display, usually a MaterialPageRoute.
        üìå Tip:
            Use Navigator.pushNamed() if you‚Äôve defined named routes in MaterialApp.
            Always use Navigator.pop(context) to return to the previous screen.
            Navigator.push() returns a Future that completes when the new route is popped ‚Äî useful for receiving results back.

    02:02:34 Pop
        Navigator.pop() is used to close the current screen (or route) and go back to the previous one.
        It‚Äôs the reverse of Navigator.push().
        When called, it pops the top route off the navigation stack.

        Navigator.pop(context);

        üß± Parameters:
            context ‚Üí The current widget‚Äôs build context.
            result (optional) ‚Üí Data you can return to the previous screen.
        üìå Tip:
            Navigator.pop() works only if a route was pushed before.
            You can return data by passing it as the second argument.
            Use it to close dialogs, modals, or bottom sheets too (not just pages).

    02:09:16 PushReplacement
        Navigator.pushReplacement() is used to navigate to a new screen while replacing the current one.
        That means the previous screen is removed from the navigation stack ‚Äî the user cannot go back using the back button.
        It‚Äôs often used for navigation flows like login ‚Üí home screen or splash ‚Üí main app.

        Navigator.pushReplacement(
            context,
            MaterialPageRoute(builder: (context) => NewScreen()),
        );
        üß± Parameters:
            context ‚Üí current widget‚Äôs BuildContext
            route ‚Üí the route (screen) to replace with
            result (optional) ‚Üí data returned to the replaced route
        üìå Tip:
            Use pushReplacement() when you don‚Äôt want users to return to the previous screen.
            For example:
            After login ‚Üí home screen
            After splash ‚Üí main app
            After onboarding ‚Üí main content
            If you need to remove all previous routes, use:
            Navigator.pushAndRemoveUntil(
                context,
                MaterialPageRoute(builder: (context) => HomeScreen()),
                (route) => false, // removes all previous routes
            );

üëæ 8. Clean UI
    02:13:18 ClipRRect
        ClipRRect is a widget that clips its child using a rounded rectangle.
        It‚Äôs commonly used to give widgets like images, containers, or cards
        rounded corners. RRect stands for ‚Äúrounded rectangle‚Äù.

        ClipRRect(
            borderRadius: BorderRadius.circular(12),
            child: Image.asset('assets/image.png'),
        );

    üß± Common properties:
        borderRadius ‚Üí controls the roundness of the corners
        clipBehavior ‚Üí defines how clipping is applied (e.g., Clip.antiAlias, Clip.hardEdge)
        child ‚Üí the widget to be clipped
    üìå Tip:
        ClipRRect only clips the visible area ‚Äî it doesn‚Äôt affect the layout.
        For rounded corners on Container backgrounds, you can also use BoxDecoration with borderRadius.
        Use ClipOval if you want to make a circle or ellipse shape instead.

    02:17:40 Hero
        Hero is a widget that provides a shared element transition between two routes (screens).
        When you navigate to another page, the Hero widget animates the transition of a matching
        widget smoothly between the screens.
        This is great for image previews, profile pictures, or cards that expand into detail pages.

        Hero(
            tag: 'hero-tag',
            child: Image.asset('assets/image.png'),
        );

        ‚ö° Important:
            Both the source and destination Hero must have the same tag.
            The tag must be unique for each Hero pair.
        üß± Common properties:
            tag ‚Üí unique identifier to match Hero widgets between screens.
            child ‚Üí the widget to animate.
            flightShuttleBuilder ‚Üí optional, allows customizing the animation look.
            transitionOnUserGestures ‚Üí allows transitions during back gesture (iOS).
        üìå Tip:
            Hero animations work best with images or simple widgets.
            Always use the same tag on both widgets.
            Works automatically with Navigator.push() and Navigator.pop().
            You can wrap any widget in a Hero‚Äînot just images!
        üß≠ Common use cases:
            Profile avatar ‚Üí profile page
            Thumbnail image ‚Üí fullscreen view
            Card preview ‚Üí detail screen

    02:22:14 SnackBar
        SnackBar is a lightweight message displayed at the bottom of the screen to provide
        brief feedback to the user. It can include a simple message, an action (like ‚ÄúUNDO‚Äù),
        and disappears automatically after a few seconds.

        ScaffoldMessenger.of(context).showSnackBar(
            SnackBar(
                content: Text('This is a SnackBar!'),
            ),
        );

        üß± Common properties:
            content ‚Üí the main widget, usually a Text.
            backgroundColor ‚Üí sets the background color of the snackbar.
            action ‚Üí adds an optional button (e.g., ‚ÄúUndo‚Äù).
            duration ‚Üí how long it stays visible.
            behavior ‚Üí e.g., SnackBarBehavior.floating to make it float above UI.
            shape ‚Üí gives rounded corners or custom shapes.
        üìå Tip:
            Always use ScaffoldMessenger.of(context) instead of the old Scaffold.of(context) (deprecated).
            SnackBar disappears automatically, but you can also dismiss it programmatically.
            Great for quick feedback like ‚ÄúSaved‚Äù, ‚ÄúCopied‚Äù, ‚ÄúDeleted‚Äù, etc.

    02:25:29 Alert - showDialog(), AlertDialog(), AboutDialog(), showAboutDialog()

        *** showDialog() is a built-in function used to display a modal dialog on top of the current screen.
        It blocks interaction with the rest of the app until the dialog is dismissed.
        It‚Äôs commonly used with widgets like AlertDialog or SimpleDialog.

        showDialog(
            context: context,
            builder: (BuildContext context) {
                return AlertDialog(
                    title: Text('Dialog Title'),
                    content: Text('This is the content of the dialog.'),
                    actions: [
                        TextButton(
                            onPressed: () => Navigator.pop(context),
                            child: Text('Close'),
                        ),
                    ],
                );
            },
        );

        üß± Common parameters:
            context ‚Üí required. The build context of the current widget.
            builder ‚Üí returns the widget to display (e.g., AlertDialog).
            barrierDismissible ‚Üí whether tapping outside closes the dialog (true by default).
            barrierColor ‚Üí color behind the dialog (e.g., semi-transparent black).
            useSafeArea ‚Üí ensures it doesn‚Äôt overlap system UI.
            useRootNavigator ‚Üí decides which navigator to use.
        üìå Tips:
            Always use Navigator.pop(context) to close the dialog.
            Return values can be passed from the dialog to the previous screen using Navigator.pop(context, value).
            You can also use showModalBottomSheet() for bottom popups.
        üß≠ Common use cases:
            Confirmation dialogs (e.g., Delete / Cancel)
            Error or success messages
            Custom forms or widgets inside a popup


        *** AlertDialog() is a material design dialog that alerts the user to a situation and provides optional
        actions (e.g., OK / Cancel). It usually contains a title, a content message, and action buttons.

        showDialog(
            context: context,
            builder: (context) {
                return AlertDialog(
                    title: Text('Alert'),
                    content: Text('This is an alert message.'),
                    actions: [
                        TextButton(
                            onPressed: () => Navigator.pop(context),
                            child: Text('Cancel'),
                        ),
                        TextButton(
                            onPressed: () {
                                // Do something
                                Navigator.pop(context);
                            },
                            child: Text('OK'),
                        ),
                    ],
                );
            },
        );

        üß± Common properties:
            title ‚Üí top of the dialog (usually Text).
            content ‚Üí main message or widget.
            actions ‚Üí list of buttons (e.g., TextButton).
            shape ‚Üí customize the border shape.
            backgroundColor ‚Üí set background color.

        *** AboutDialog() is a prebuilt dialog that displays information about the app, such as its name,
        version, icon, and legal info. It‚Äôs typically used in ‚ÄúAbout‚Äù sections of apps.

        showAboutDialog(
            context: context,
            applicationName: 'My Flutter App',
            applicationVersion: '1.0.0',
            applicationIcon: Icon(Icons.info),
            children: [
                Text('This is a simple Flutter app.'),
            ],
        );
        üß± Common properties:
            applicationName ‚Üí app‚Äôs name.
            applicationVersion ‚Üí version number.
            applicationIcon ‚Üí icon widget shown at the top.
            applicationLegalese ‚Üí legal text (e.g., copyrights).
            children ‚Üí list of extra widgets or info.

        üìå Tip:
            AlertDialog is used for actions and confirmations.
            AboutDialog is used for static info about the app.
            Both appear on top of the UI and block interaction until dismissed.
